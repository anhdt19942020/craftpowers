## Trigger wiring

Trigger mechanism = **Hybrid** (auto via skill description matching + hard-wired references trong subagent prompt template + slash command).

### A) `skills/subagent-driven-development/implementer-prompt.md`

Thêm block trước "Begin task":

```markdown
## Conditional skill loads

Before implementing, check task spec:

- **Đụng framework/library bên ngoài** (import, npm install, lib lạ)?
  → Load skill `source-driven-development`. Verify từng API call qua context7 MCP. Cite docs trong commit.

- **File sửa có extension `.tsx/.jsx/.vue/.svelte/.css/.scss`** hoặc spec đề cập "UI/component/page/layout"?
  → Load skill `frontend-ui-engineering`. Theo component arch, state mgmt, responsive, WCAG 2.1 AA.

- **Spec đề cập "API/endpoint/route/REST/GraphQL/RPC"** hoặc tạo handler file (`routes/`, `api/`, `controllers/`, `*.endpoint.*`)?
  → Load skill `api-and-interface-design`. Theo contract-first, versioning, error shape.
```

### B) `skills/subagent-driven-development/code-quality-reviewer-prompt.md`

Thêm block load on demand:

```markdown
## Reference checklists (conditional)

Sau khi review code, load checklist phù hợp diff:

- **Diff có FE file** (`.tsx/.jsx/.vue/.svelte`) → load `references/accessibility-checklist.md`. Check WCAG.
- **Diff đụng auth/input/network/storage** → load `references/security-checklist.md`. OWASP Top 10.
- **Diff đụng hot path / build size / DB query** → load `references/performance-checklist.md`. Core Web Vitals + backend.
- **Diff thêm/sửa test** → load `references/testing-patterns.md`. Structure, naming, mocking.

Find ≥1 issue per loaded checklist HOẶC explicit pass với lý do.
```

### C) `commands/man-quick.md`

Thêm sau "Verify (test/lint/typecheck)":

```markdown
## Verify (extended)

- Standard: test/lint/typecheck pass.
- **Nếu file sửa là FE** (`.tsx/.jsx/.vue/.svelte`):
  → Load skill `browser-testing-with-devtools`. Bật dev server, dùng Chrome DevTools MCP check DOM/console/network. Report runtime evidence.
  → Nếu MCP không available → skip với note "browser MCP unavailable, manual verify needed".
```

### D) `skills/test-driven-development/SKILL.md`

Thêm 1 line ở phần "Test structure":

```markdown
> **Reference:** `references/testing-patterns.md` — chi tiết pattern test (structure, naming, mocking, React/API/E2E examples, anti-patterns).
```

### Trigger flow

```
User → implementer subagent
  → đọc task spec
  → match conditional rules ở (A)
  → load skill tương ứng (1 hoặc nhiều, không xung đột)
  → implement theo skill rules

User → code-quality-reviewer subagent
  → đọc diff
  → match conditional rules ở (B)
  → load checklist tương ứng
  → review + report (≥1 issue per checklist hoặc explicit pass)

User → /man-quick
  → quick-fix agent
  → verify standard
  → nếu FE → browser-testing skill (C)
```

### Edge cases

- Multi-skill match (FE file + đụng API) → load cả 2 skill, không xung đột vì scope khác.
- MCP unavailable (no Chrome DevTools / no context7) → skip skill, log warning, không block work.
- Skill description mismatch user query → fallback về hard-wired rules trong prompt template (không miss).
