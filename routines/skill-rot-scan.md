# Skill Rot Scanner

Scan all skills and agents for stale references. Open a GitHub issue if any are found. If nothing is found, do nothing.

## Steps

1. **Collect files to scan:**
   - All `skills/**/SKILL.md` files
   - All `agents/*.md` files

2. **Search for stale patterns in each file:**

   **Old model IDs** (grep for any of these):
   - `claude-3[^-.]` or `claude-3\.` (Claude 3 era)
   - `claude-3\.5` or `sonnet-3\.5` or `claude-3-5`
   - `claude-opus-4-5` or `claude-sonnet-4-5` or `claude-haiku-4-5`
   - `claude-opus-4-0` or `claude-sonnet-4-0`

   **Broken internal links** — for every reference of the form `[[...]]` or `references/<filename>` found in a file:
   - Check if the target file exists on disk relative to the skill's directory
   - A reference is broken if the file does not exist

   **Removed providers** (grep for any of these words, case-insensitive):
   - `cursor` (as a provider/IDE)
   - `gemini` (as a provider)
   - `codex` (as a provider)
   - `opencode` (as a provider)
   - Match only when used as a provider name — ignore legitimate uses (e.g. "cursor position")

3. **Collect all findings** into a structured list:
   ```
   - [FILE] <relative-path>: <category> — <exact match or broken link>
   ```

4. **If no findings:** Stop. Do not create an issue. Do not comment anywhere.

5. **If findings exist:** Create a GitHub issue:
   ```
   gh issue create \
     --title "Skill rot detected — $(date +%Y-%m-%d)" \
     --label "skill-rot" \
     --body "<findings list>"
   ```

   Issue body format:
   ```
   ## Skill Rot Report — <date>

   The weekly skill-rot-scan routine found the following stale references:

   ### Stale Model IDs
   - ...

   ### Broken Internal Links
   - ...

   ### Removed Provider References
   - ...

   ---
   *Generated by `routines/skill-rot-scan.md`*
   ```

   Only include sections that have findings.

## Notes

- Run from the repository root.
- `gh` CLI must be authenticated.
- The `skill-rot` label must exist on the repo (create it if missing: `gh label create skill-rot --color "#e11d48"`).
- Do not open duplicate issues: check `gh issue list --label skill-rot --state open` first. If an open issue already exists from this week, append a comment instead of opening a new issue.
