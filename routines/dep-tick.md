# Dependency Tick

Check for outdated dependencies. Open a GitHub issue only for **major** version bumps — those require human review. Skip minor and patch updates (too noisy).

## Steps

1. **Check for Node.js dependencies** (only if `package.json` exists in repo root):
   ```
   npm outdated --json
   ```
   Parse the JSON output. For each package, determine if the available version is a **major** bump relative to the current version (i.e. `major(available) > major(current)`).

2. **Check for Python dependencies** (only if `requirements.txt` exists in repo root):
   ```
   pip list --outdated --format=json
   ```
   Parse the JSON output. For each package, compare current vs latest version to determine if it is a **major** bump.

3. **Group findings:**
   - Collect only **major** bumps from both sources.
   - Skip minor and patch updates entirely.

4. **If no major bumps found:** Stop. Do not create an issue.

5. **If major bumps found:** Create a GitHub issue:
   ```
   gh issue create \
     --title "Major dependency updates available — $(date +%Y-%m-%d)" \
     --label "dependencies" \
     --body "<findings>"
   ```

   Issue body format:
   ```
   ## Major Dependency Updates — <date>

   The following packages have **major** version updates available and require human review before upgrading:

   | Package | Current | Available | Ecosystem |
   |---------|---------|-----------|-----------|
   | <name>  | <cur>   | <new>     | npm / pip |

   Minor and patch updates were skipped.

   **Next step:** Review breaking changes for each package before upgrading.

   ---
   *Generated by `routines/dep-tick.md`*
   ```

## Notes

- Run from the repository root.
- `npm` and `pip` must be available in the routine environment as applicable.
- `gh` CLI must be authenticated.
- The `dependencies` label must exist (create if missing: `gh label create dependencies --color "#0075ca"`).
- Do not open duplicate issues: check `gh issue list --label dependencies --state open` first. If one already exists from this week, skip.
